სარეკლამო კომპონენტის სრული ტესტი

📅 3 მარტი, 2026  |  ✍️ Test Author  |  desktop ≥ 1281px mobile ≤ 1280px

ეს გვერდი შექმნილია სარეკლამო კომპონენტის ყველა ფუნქციის სრული ტესტირებისთვის. გვერდი შეიცავს საკმარის კონტენტს სქროლის გასატესტად, ჰედერი დაფიქსირებულია ზევით, ფუთერი კი ქვევით — სტიკი ბანერის ჩამალვის სცენარი ამ ბოლო ელემენტის visibility-ით მოქმედებს.

🧪 რა იტესტება?

1. Preloader — 45 წამიანი cooldown-ით, 30%-ის დაბლა random skip-ით. მობილური-მხოლოდ (≤1280px). Reset ღილაკი ზედა მარცხენა კუთხეში localStorage-ს ასუფთავებს.

2. Sticky Banner — 3 წამის შემდეგ ან 100px+ სქროლის დროს ჩნდება. Desktop-ზე purple sticky, mobile-ზე dark pink. Close ღილაკი და swipe/flick gesture.

3. Static Ads — Leaderboard (mobile), Article Top/Bottom (desktop/mobile), Sidebar (desktop). Banner Cleanup 17 წამის შემდეგ broken ads-ს ასუფთავებს.

4. Footer Intersection — footer-ს მიახლოვებისას sticky ქრება, დაბრუნებისას ჩნდება.

📜 სტატია — სქროლის ტესტი

ლორემ იფსუმ სტილის ქართული ტექსტი. ახალი ამბები, რეპორტაჟები, ანალიზი — ეს ყველაფერი ქმნის გვერდის სიგრძეს, რომელიც საშუალებას გვაძლევს ვნახოთ, თუ როგორ იქცევა სარეკლამო კომპონენტი სკროლის სხვადასხვა ეტაპზე.

საქართველოს ეკონომიკა, პოლიტიკა, კულტურა და სხვა სფეროები — ყოველი მათგანი ქმნის საინტერესო კონტენტს, რომელიც ამ გვერდზე სიმულირებულია. მკითხველი სქროლავს ქვევით და სარეკლამო ბანერები გამოჩნდება სწორ მომენტში.

🔧 ტექნიკური დეტალები

Sticky rotation: sessionStorage ინახავს ბოლო ნაჩვენები sticky-ის ინდექსს — შემდეგ ჯერზე მომდევნო გამოჩნდება. ორი sticky გვაქვს: purple (#7B2D8B, 130px) და dark pink (#C2185B, 120px).

Preloader cooldown: localStorage-ში ინახება ბოლო ჩვენების timestamp. 45 წამის გასვლამდე ახალი preloader არ გამოჩნდება. Reset ღილაკი ამ timestamp-ს შლის.

Banner cleanup: 17 წამის შემდეგ, DOMContentLoaded-დან, სკრიპტი ამოწმებს ყველა [data-banner-wrapper]-ს. თუ შიგთავსი ცარიელია (img load failed, iframe არარის), ელემენტი remove() ხდება.

📱 Mobile vs Desktop

Breakpoint: 1281px. Desktop-ზე sticky desktop-ის bannerWrapper-ში იტვირთება, mobile-ზე კი — mobile-ში. Mobile-ზე sticky ელოდება preloaderClosed event-ს (ან 10 წამის safety timer-ს) სანამ გამოჩნდება.

iOS-ზე touchstart/touchend listeners გამორთულია (isIOS check) — Safari-ს გამო. Wheel event desktop flick-ს ატრიგერებს deltaY > 200 ზე.

🎨 Flick Animation

Desktop: wheel event-ზე banner flick-up ან flick-down კლასი ემატება 500ms-ით, შემდეგ იხსნება. Mobile: scroll direction ამოწმება — შეპასუხება 250ms stop-ის შემდეგ ხდება.

close-trigger SVG-ს blowShrink animation აქვს: scale(1) → scale(0.7) + opacity 0, 1.5 წამი. შემდეგ banner removing კლასი ემატება და 0.8 წამში ქრება.

⚠️ კოდში ნაპოვნი დაკვირვებები

✅ კარგი: autoShowTimer closure სწორადაა გამოყენებული — closeBanner-ს clearTimeout-ამდე წვდომა აქვს სხვა ფუნქციის scope-დან. onScrollTick ლეზი ვარიაბლი სწორად გადაეცემა removeEventListener-ს.

⚠️ Minor: localStorage.setItem preloader hideP-ში try/catch-გარეშეა — private browsing-ში exception-ი გაისვრება, მაგრამ მხოლოდ კონსოლში, UI არ მოიტეხება.

⚠️ Minor: renderAd PHP ფუნქცია !function_exists-ით ზომავს — Blade-ის parallel rendering-ში safe, მაგრამ HTML ტესტში ეს ლოგიკა JS-ად გადაიწერა.

✅ Solid: IntersectionObserver footer disconnect closeBanner-ში სწორად ხდება memory leak-ის გარეშე. isMoving/isClosing flags ორმაგ trigger-ს კარგად ბლოკავს.

📊 დამატებითი ტექსტი სქროლისთვის

ეს პარაგრაფი არსებობს მხოლოდ იმისთვის, რომ გვერდი საკმარისად გრძელი იყოს. სარეკლამო კომპონენტი 3 წამის შემდეგ გამოჩნდება — ასე რომ, თუ გვერდი ძალიან მოკლეა, sticky-ს ჩვენება ვერ გაიტესტება სრულად.

დამატებითი ტექსტი: კვლევები გვიჩვენებს, რომ მომხმარებელი ვებ-გვერდზე საშუალოდ 3-5 წამს ატარებს სანამ გადაწყვეტს დარჩეს თუ გადაიდგამს. სწორედ ამიტომ sticky ბანერი 3 წამის შემდეგ ჩნდება — ოპტიმალური მომენტი engagement-ისთვის.

კიდევ ერთი პარაგრაფი. ვებ-რეკლამის ინდუსტრია მილიარდობით დოლარის ბრუნვა ქმნის ყოველწლიურად. Display advertising, native ads, programmatic buying — ყველა ეს ფორმატი სხვადასხვა UX challenge-ს ქმნის. ჩვენი კომპონენტი ერთ-ერთი კარგი მიდგომაა ამ challenge-ის გადასაჭრელად.

Intersection Observer API-ს გამოყენება footer-ის დასაფიქსირებლად ბევრად ეფექტურია, ვიდრე scroll event-ზე getBoundingClientRect() — CPU load გაცილებით ნაკლებია. ეს კარგი პრაქტიკაა.

requestAnimationFrame double-wrap showNow()-ში (rAF inside rAF) უზრუნველყოფს, რომ display:flex სეტი და opacity transition ერთ paint cycle-ში არ მოხვდეს — ეს prevents FOUC (flash of unstyled content).

🔚 დასკვნა

ეს სატესტო გვერდი სრულად ასახავს პროდაქშენ გარემოს: ფიქსირებული ჰედერი, გრძელი სტატია, sidebar, footer — ყველაფერი ერთად. სარეკლამო კომპონენტის ყველა ქვე-სისტემა ტესტირდება.
